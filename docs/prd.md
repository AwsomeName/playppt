# PPT 讲解 Agent PRD

## 1. 文档信息

- 产品名称：PPT 讲解 Agent
- 所属项目：`play-ppt`
- 文档版本：v0.1
- 目标版本：MVP

## 2. 背景与目标

### 2.1 背景
传统 PPT 演示依赖人工讲解，存在讲解质量不稳定、无法 7x24 小时服务、互动成本高的问题。通过引入语音与大模型能力，可以实现“自动讲解 + 实时问答”的演示体验。

### 2.2 目标
- 在浏览器内完成 PPT 自动播放与讲解。
- 支持用户语音提问，并基于 PPT 内容回答。
- 在讲解过程中可被语音打断并恢复。

### 2.3 成功指标（MVP）
- 演示成功率 >= 95%（从开始到结束无致命中断）。
- 语音问答平均首响应时间 <= 5s。
- 与 PPT 内容相关的问题，回答可用率 >= 80%（人工评估）。

## 3. 用户故事

- 作为销售，我希望系统自动讲解方案 PPT，减少重复劳动。
- 作为听众，我希望用语音提问，并立刻得到与当前页相关的回答。
- 作为运营，我希望查看问答和讲解日志，以优化内容。

## 4. 功能需求

## 4.1 PPT 播放与控制

### FR-001 打开演示
- 输入：PPT 资源地址或预置演示 ID。
- 输出：在浏览器中打开可播放页面并进入准备状态。

### FR-002 翻页控制
- 支持：下一页、上一页、跳转第 N 页、暂停自动翻页、恢复自动翻页。
- 来源：系统自动调度或用户语音命令。

### FR-003 页面状态同步
- 实时维护当前页索引、总页数、页面标题（若可提取）。

## 4.2 讲解能力

### FR-010 讲解词管理
- 每页可配置讲解词文本。
- 若缺少讲解词，提供默认模板讲解（基于页面提取内容生成）。

### FR-011 语音播报
- 讲解词通过 TTS 播放。
- 支持语速、音色参数配置。
- 播放期间可被用户打断。

### FR-012 讲解流程控制
- 页面进入后自动触发讲解。
- 讲解结束后执行翻页策略（自动/手动确认）。

## 4.3 语音问答能力

### FR-020 语音识别
- 用户语音输入实时或分段转文本。
- 识别失败时给出重试提示。

### FR-021 意图识别
- 区分：问答意图、翻页命令、控制命令（暂停/继续）。

### FR-022 PPT 问答
- 优先检索当前页，再检索全局页面内容。
- 输出回答时尽量简洁并可附“来源页码”。
- 当信息不足时明确说明“不确定”，避免编造。

### FR-023 讲解恢复
- 回答完成后自动回到讲解流程（同页继续或按策略翻页）。

## 4.4 记录与管理

### FR-030 日志记录
- 记录关键事件：翻页、讲解开始/结束、问题与回答、错误信息。

### FR-031 会话追踪
- 每场演示有唯一会话 ID，可用于追溯与分析。

## 5. 非功能需求

- **性能**：控制指令响应 < 500ms（本地网络环境）。
- **稳定性**：单场 1 小时演示内无崩溃。
- **可用性**：异常时可恢复到“仅手动翻页”模式。
- **安全性**：语音与文本日志可按配置脱敏存储。

## 6. 约束与假设

- 假设 PPT 可被转换为可在浏览器中播放的形态。
- 假设语音输入设备可用，浏览器具备麦克风权限。
- 初期仅支持中文问答与中文语音播报。

## 7. 验收标准（MVP）

- 可完成一场 10 页以上 PPT 的自动讲解。
- 用户可通过语音控制至少 4 类命令：下一页、上一页、跳页、暂停/继续。
- 用户语音提问后可得到与 PPT 相关的文本与语音回答。
- 全流程有完整日志，支持问题追踪。

## 8. 不做范围（MVP）

- 多人并发同屏互动管理。
- 跨多份文档联合问答。
- 复杂图表语义深度解析（仅做基础文本理解）。
