# PPT 讲解 Agent 产品文档

> 工程实现、控制接口与状态机、里程碑验收以 `docs/ai-dev-plan.md` 为准。整体架构、**可承诺的控制动作清单**、口播与问答数据、章节与降级策略的统一定义见该文档**第 1.1 节**。

## 1. 产品定位

`play-ppt` 是一个基于 Web-PPT 播放环境的智能讲解助手。它通过浏览器加载并播放 PPT，自动控制翻页与讲解，并支持语音问答，让用户在演示过程中可以“边看边问”。

产品核心价值：
- 将“静态 PPT”升级为“可对话的讲解体验”。
- 降低演讲门槛，让非专业演讲者也能完成高质量讲解。
- 提升听众参与度，支持实时提问与即时回答。

## 2. 目标用户

- 销售与售前人员：产品演示、方案宣讲。
- 教育与培训人员：课程讲解、知识点问答。
- 企业内部汇报人员：汇报自动化讲解。
- 展会/展厅场景：无人值守讲解终端。

## 3. 典型场景

1. **自动讲解模式**
   - 用户上传或选择一份 PPT。
   - Agent 按页面顺序播放并朗读讲解词。
   - 每页可设停留时长与自动翻页策略。

2. **互动问答模式**
   - 听众通过语音提出问题。
   - Agent 结合当前页和整份 PPT 内容回答。
   - 回答后自动回到讲解流程。

3. **主持辅助模式**
   - 主讲人手动控制翻页，Agent 提供讲稿补充与问答支持。
   - 支持「下一页/上一页/到第 N 页/开始/暂停/继续/结束」等语音命令，并归一到有限 `action` 集（以 `ai-dev-plan` 第 1.1.5 节为准）；**「到第 X 章」**仅当演示数据提供章节-页码映射时作为稳定控制项。

## 4. 核心能力

- **Web-PPT 播放控制**
  - 打开演示链接
  - 控制翻页、跳页、暂停/恢复
  - 获取当前页状态

- **讲解词播放**
  - 每页关联讲解文本
  - TTS 语音合成并播放
  - 支持语速、音色、音量配置

- **语音理解与问答**
  - ASR 语音转文本
  - 意图识别（提问、翻页命令、打断控制）
  - 基于 PPT 内容的检索增强回答（RAG）

- **会话管理**
  - 保持上下文（当前页、历史问题）
  - 控制回答范围优先关联当前页内容
  - 提供“不确定时的安全回复”

## 5. 产品边界

- 当前仅面向 Web 环境中的 PPT 播放与互动。
- 默认聚焦单份 PPT 的问答，不处理跨文档知识库融合（可作为后续版本能力）。
- 不承诺所有开放性问题都能给出准确答案；无法确认时应明确说明并引导查看页面内容。

## 6. 非功能要求

- **实时性**：语音提问后 2-5 秒内给出首句反馈。
- **稳定性**：长时间演示（30-60 分钟）不中断。
- **可观测性**：记录翻页、问答、错误日志，便于排查与复盘。
- **可扩展性**：后续可接入多种 Web-PPT 引擎与多语言语音能力。

## 7. 版本规划

- **MVP（V0.1）**
  - Web-PPT 基础播放控制
  - 每页讲解词播放
  - 基于 PPT 文本的语音问答

- **V0.2**
  - 提升问答准确性（页面级检索 + 关键信息高亮）
  - 语音命令增强（更自然的指令表达）
  - 多套讲解风格配置（正式/简洁/营销）

- **V0.3**
  - 多文档场景支持
  - 演讲数据看板（提问热点、停留时长、讲解完成率）
  - 企业级部署与权限管理
